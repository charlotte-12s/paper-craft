# ICSE — IEEE/ACM International Conference on Software Engineering

## Basic Info
- Full name: IEEE/ACM International Conference on Software Engineering
- CCF Level: A
- Acceptance rate: ~18% (2024: ~18%)
- Page limit: 10 pages + supplementary
- Deadline: Typically October (notification January)
- Double-blind review: Yes

## Review Weights
- Innovation: 20% — Engineering rigor paramount
- Empirical: 35% — Empirical study design, statistical validity
- Practical: 25% — Practical adoption and tool usage
- Theory: 5% — Not primary criterion
- Clarity: 15% — Threats to validity must be addressed

## Writing Style
- Title: SE task + method
- Abstract: SE problem → study design → findings → practical implications
- Introduction: Practical SE motivation with industry relevance
- Experiments: Empirical study design with statistical validity, threats to validity section

## Recent Trends (2024-2026)
- LLM for software engineering
- Automated program repair
- DevOps and AIOps
- Code generation and understanding
- Software testing automation

## Reviewer Common Concerns
- "Threats to validity not addressed"
- "Toy project without real-world validation"
- "Study design has confounding variables"
- "Statistical analysis is insufficient"

## Anti-patterns
- Toy project without real-world validation
- No threats to validity section
- Weak empirical methodology
- Small sample size without justification

## Success Patterns
- Industrial case study + tool adoption
- Rigorous empirical methodology with statistical validity
- Real-world SE problem with practical solution
- Tool that is adopted in practice
