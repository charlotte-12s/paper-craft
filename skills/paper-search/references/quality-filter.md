# Paper Quality Filtering Standards

## Quality Tiers

### ⭐⭐⭐ Must Read
- Citation count: >500 (or >100 if published <2 years ago)
- Has open-source code on GitHub
- Published at top venue (CCF-A or equivalent)
- Reproducible results (code + data + seeds available)

### ⭐⭐ Should Read
- Citation count: >100 (or >30 if published <2 years ago)
- Has open-source code OR detailed pseudocode
- Published at recognized venue (CCF-A/B or equivalent)
- Results are reproducible from description

### ⭐ Background Reference
- Published but limited impact
- May lack code or have incomplete implementation
- Useful for context, methodology borrowing, or gap identification

### ❌ Skip
- Preprint only with no code and low citations
- Non-reproducible (no code, no data description)
- Withdrawn or superseded by later version

## Filtering Rules

1. Always prefer papers with code over papers without code
2. Prefer conference papers over journal papers (faster review, more recent)
3. Prefer papers with >10 citations over papers with <10 citations
4. Prefer papers from target conference (for taste matching)
5. If a paper has both arXiv and published versions, cite the published version
6. Check if code repository is actively maintained (last commit <6 months)
