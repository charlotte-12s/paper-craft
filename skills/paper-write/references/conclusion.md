# Conclusion Writing Guide

## Structure (3 paragraphs)

### Paragraph 1: Contributions
- Restate the main contribution (echo introduction, don't repeat verbatim)
- Summarize key results with numbers
- State what this enables

### Paragraph 2: Limitations
- Be honest but not self-defeating
- Frame as "future work needed" not "our paper is bad"
- 2-3 specific, constructive limitations
- Example: "Our method currently requires [resource], which may limit applicability in [setting]. Exploring [direction] could address this."

### Paragraph 3: Future Work
- 2-3 concrete, actionable directions
- Each should build on your contribution
- Example: "Applying [method] to [new domain] is a promising direction, as [reason]."

## Common Mistakes
- Repeating the abstract → Conclusion should add reflection, not copy
- Listing all contributions again → Focus on the MAIN one
- Overclaiming → "We solve X" → "We present an approach that significantly improves X"
- Ignoring limitations → Reviewers will find them anyway; address them first
- Vague future work → "We plan to explore more" → "We plan to extend to [domain] by [approach]"
